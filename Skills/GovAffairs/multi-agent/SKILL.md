# 系统架构

```
用户 → 主Agent（调度） → 子Agent → 审核/润色 → 输出
```

## 子Agent

| Agent | 职责 | 文件 |
|-------|------|------|
| 简报Agent | 生成简报 | baoji-daily-briefing |
| 讲话稿Agent | 生成讲话稿 | speech-writer |
| 调研报告Agent | 生成报告 | research-report |
| 审核Agent | 审核质量 | agents/review.md |
| 润色Agent | 去除AI味 | agents/polish.md |

## ⚠️ 强制规则

**调用子Agent时，必须读取对应文件作为prompt，禁止自己编写**

# 调度流程

## 简报
```
确认 → 简报Agent → 审核Agent → 输出
```

## 讲话稿
```
确认（主题/风格/素材）→ 讲话稿Agent → 审核Agent → 润色Agent → 输出
```

## 调研报告
```
【必须先确认】→ 确认（主题/类型/版本）→ 调研报告Agent → 审核Agent → 润色Agent → 输出
```

### 调研报告确认步骤（必须执行）
当用户要求生成调研报告时，必须先询问：
1. **调研主题**是什么？（如：乡村振兴、营商环境等）
2. **报告版本**选哪个？
   - 决策版（3000-5000字）- 领导决策参考
   - 执行版（8000-12000字）- 部门执行方案
   - 学术版（10000-15000字）- 学术研究
   - 汇报版（PPT大纲+讲稿）- 汇报演示
3. **调研范围**是？（如：宝鸡市、陕西省等）

确认以上信息后再调用调研报告Agent

## ⚠️ 关键规则

**生成Agent完成后，必须自动调用审核Agent，不能跳过**

## 审核失败
打回重做，最多3次
