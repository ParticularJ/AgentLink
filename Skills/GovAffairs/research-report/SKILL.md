# 政务调研报告 Skill

## 任务理解

用户需要生成一份政务调研报告，包含以下步骤：
1. 确认报告类型（决策版/执行版/学术版/汇报版）
2. 搜索相关素材（政策、新闻、数据、案例）
3. 如用户提供了素材，整合用户素材 + 搜索素材
4. 按指定类型生成报告
5. 输出为 Word 文档（.docx）

## 报告类型与规格

| 类型 | 字数 | 结构要求 |
|-----|------|---------|
| 决策版 | 3000-5000 | 四层逻辑（简略） |
| 执行版 | 8000-12000 | 四层逻辑（详细） |
| 学术版 | 10000-15000 | 四层逻辑 + 学术规范 |
| 汇报版 | PPT大纲+讲稿 | 四层逻辑 + 演示结构 |

## 统一框架：四层逻辑

**所有类型都必须包含以下四层逻辑：**

### 第一层：战略定位（为什么做）
- 政策背景（中央/省级/市级要求）
- 现实需求（痛点、问题、差距）
- 战略意义（价值、目标）

### 第二层：现状诊断（现在怎样）
- 已有成效（成绩、亮点）
- 存在问题（短板、瓶颈）
- 原因分析（深层原因）

### 第三层：对标研究（别人怎么做）
- 先进地区/部门做法
- 成功案例拆解
- 可借鉴经验

### 第四层：路径设计（我们怎么做）
- 总体思路
- 具体举措
- 保障机制
- 实施步骤

### Step 6: 输出

## 注意事项

1. **决策版要简洁**：领导时间宝贵，直接给结论和建议
2. **执行版要详细**：业务部门需要可操作的具体措施
3. **学术版要规范**：需要有理论框架和文献支撑
4. **汇报版要生动**：PPT 要图文并茂，讲稿要口语化
5. **数据必须标注来源**：政府报告，数据来源是底线
6. **禁止糊弄**：搜不到的数据写"【请补充具体数据】"
